**12/12**

오늘 알아보다 폰트나 font icon에 색이 빠지는 현상이 벡터화에서 일어난다는 것을 알았다.

이것을 이용해 벡터화에서  다시 디자인을 해야하는 데 이러한 방법 말고도

다른 방법을 찾았다.

이미지의 색상 보존을 시키는 방법이다.

방법은 GIMP를 이용해서 색상을 보존한 채로 svg파일을 만드는 것인데, 가능성이 있다고 생각한다.

이미지 색상 정보를 보존할 수 있다면 font maker에서 font를 만드는 과정에서 사진을 svg로 만들고 svg를 ttf파일로 만드는 방식이라면

font maker를 사용할 수 있다.

하지만 font maker가 검은색을 제외한 모든 색상을 검은색으로 만드는 형식의 시스템이라면 어쩔 수 없이 Inkscape를 사용해 jpg를 svg로 만들어 svg를 ttf로 만드는 수 밖에 없다.

하지만 아마 벡터화를 하면서 단순히 색이 사라진거라고 추측 중이다.

- GIMP -> font maker 

- GIMP -> Inkscape -> font icon으로

2가지 방법이 있는데 1은 ttf파일 폰트이고

2는 font icon이라는 점에 차이가 있다.

일단 1번을 시도해봐야 겠다.

**12/13**

굉장히 많은 일이 있었다.

많은 프로그램을 써보고 많은 난관이 있었다.

GIMP와 inkscape를 이용해 색상 보존과 벡터화를 진행하려 했다.

하지만 색상 보존은 GIMP로 하기에는 gimp는 너무 불친절한 프로그램이다.

나는 Vectr라는 사이트로 이미지를 svg로 convert하였는데 문제가 생겼다.

이제 fontforge로 svg로 ttf를 만들려고 하는 데 너무 복잡하거나 올바르지 않습니다. 라는 문구가 떴다.

그래서 간소화를 하려는 데 간소화 할 것이 없다.

그렇다면 올바르지 않다는 것인가? 지금 상태에서는 잘 모르겠다.

그래서 calligraphr로 템플릿에 svg를 넣을까 생각하는데 이미 벡터화가 된 svg파일을 바로 인식해서 그냥 ttf파일로 변환할지 아니면 벡터화를 다시 한 번해서 색이 지워질 지 고민이다.

그리고 참고 : figma로 디자인을 했을 때 mask기능을 해 디자인을 하였다면 fimga export기능을 쓸 수 없다. svg로 만들면 인식했을 때 mask로 다듬은 모양이 나오지 않기 때문이다.

이 기능은 아직 고쳐지지 않았다.

SVGO같은 사이트는 또한 404에러가 뜨는 걸 보니 사라진 것 같기도 하다.

참고 : svg파일은 xml로 되어있다.

vectr라는 컨버트 사이트를 이용해 jpg를 svg로 만들었다.
